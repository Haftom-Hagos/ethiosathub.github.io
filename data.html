This section provides administrative boundary datasets of Ethiopia in GeoPackage format. The datasets are organized into different administrative levels:

  • Level 0 – National boundary of Ethiopia
  • Level 1 – Regional boundaries
  • Level 2 – Zonal boundaries
  • Level 3 – Woreda (district) boundaries
  • Level 4 – Kebele (local administrative unit) boundaries

Each dataset is available in two coordinate reference systems:

  • Geographic Coordinate System (GCS): EPSG:4326 – WGS 84
  • Projected Coordinate System (PCS): EPSG:32637 – WGS 84 / UTM Zone 37N

The GCS version is useful for web mapping and global datasets, while the PCS version is recommended for spatial analysis and distance/area measurements. These datasets are suitable for GIS applications, research, and visualization projects related to Ethiopia’s administrative divisions.

GeoPackage Format: GeoPackage (GPKG) is an open, internationally recognized format for storing and sharing geographic data. GeoPackage is supported by most modern GIS software including QGIS, ArcGIS, and GDAL. It allows users to organize multiple layers and their attributes efficiently in one file, making it straightforward to exchange and reuse data. In geopackage format both vector and raster layers can be stored in a single, compact file. Using GeoPackage is a normal part of today’s geospatial workflows, offering a reliable and future-proof way to manage spatial information.
