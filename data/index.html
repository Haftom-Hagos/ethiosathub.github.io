This section provides administrative boundary datasets of Ethiopia in Shapefile, GeoJSON, and GeoPackage formats. The datasets are organized into different administrative levels:

  • Level 0 – National boundary of Ethiopia
  • Level 1 – Regional boundaries
  • Level 2 – Zonal boundaries
  • Level 3 – Woreda (district) boundaries

Each dataset is available in two coordinate reference systems:

  • Geographic Coordinate System (GCS): EPSG:4326 – WGS 84
  • Projected Coordinate System (PCS): EPSG:32637 – WGS 84 / UTM Zone 37N

Shapefile Format (Zipped)

Shapefile is one of the most widely used vector formats in GIS. It consists of multiple files (.shp, .shx, .dbf, .prj, etc.), so here they are provided as .zip archives for easy download. Widely supported by GIS software, but has size and attribute limitations.

GeoJSON Format

GeoJSON is a lightweight JSON-based format for geographic data. Ideal for web mapping, APIs, and visualization in browsers. Supports geometry, features, and collections.

GeoPackage Format

GeoPackage (GPKG) is an open, compact format for storing vector and raster layers in one file. Supported by modern GIS software like QGIS, ArcGIS, and GDAL. Recommended for reliable and future-proof data exchange.
