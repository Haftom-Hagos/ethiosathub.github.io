This section provides administrative boundary datasets of Ethiopia in Shapefile, GeoJSON, and GeoPackage formats. The datasets are organized into different administrative levels:

  • Level 0 – National boundary of Ethiopia
  • Level 1 – Regional boundaries
  • Level 2 – Zonal boundaries
  • Level 3 – Woreda (district) boundaries

Each dataset is available in two coordinate reference systems:

  • Geographic Coordinate System (GCS): EPSG:4326 – WGS 84
  • Projected Coordinate System (PCS): EPSG:32637 – WGS 84 / UTM Zone 37N

Click the links below to download the data:

Shapefile Format (Zipped)

Shapefile is one of the most widely used vector formats in GIS. It consists of multiple files (.shp, .shx, .dbf, .prj, etc.), so here they are provided as .zip archives for easy download. Shapefiles are supported by almost all GIS software, but note that they have size and attribute limitations compared to newer formats.

GeoJSON Format

GeoJSON is a lightweight format based on JSON for representing geographic data. It is especially suitable for web mapping applications and APIs. GeoJSON supports encoding of geometry, features, and feature collections, making it ideal for visualization in web browsers and sharing over the internet.

GeoPackage Format

GeoPackage (GPKG) is an open, internationally recognized format for storing and sharing geographic data. It supports both vector and raster layers in a single file, making it compact and efficient. GeoPackage is supported by modern GIS software including QGIS, ArcGIS, and GDAL, and is recommended for reliable, future-proof data exchange.
